# MVP 规划：离线零售门店销售开单系统

## 项目背景

### 需求分析
- **场景**：零售门店（个体户）
- **用户**：1人使用（店主）
- **环境限制**：
  - 无网络或网络不稳定
  - 无额外供电（需考虑电池续航）
  - 使用手机或平板（iOS/Android）
- **核心痛点**：现有台湾本土方案要么需要网络，要么功能过于复杂

### 目标
打造一个**极简、纯离线、手机优先**的销售开单工具，让个体户能快速上手，在断网环境下也能正常经营。

## 技术方案

### 推荐技术栈

| 层级 | 方案 | 理由 |
|-----|------|------|
| **前端框架** | React + TypeScript | 生态成熟，PWA支持好 |
| **构建工具** | Vite | 快速、PWA插件完善 |
| **UI样式** | Tailwind CSS | 轻量、可定制 |
| **状态管理** | Zustand | 简单、轻量 |
| **本地存储** | IndexedDB (Dexie.js) | MVP快速开发 |
| **未来升级** | SQLite WASM + OPFS | 大容量、完整SQL能力 |

### 为什么选 PWA 而非原生 App？

1. **开发效率**：一套代码同时支持 iOS/Android
2. **离线能力**：Service Worker 实现纯离线使用
3. **分发简单**：无需上架 App Store，扫码即用
4. **更新方便**：自动更新，无需用户手动下载

### SQLite WASM 评估

参考：https://developer.chrome.com/blog/sqlite-wasm-in-the-browser-backed-by-the-origin-private-file-system

| 优势 | 劣势 |
|-----|------|
| 大容量存储（100MB+） | 包体积 +1~2MB |
| 完整 SQL 查询能力 | 浏览器支持要求较高 |
| 备份直接下载 .db 文件 | 学习成本稍高 |
| 复杂报表查询方便 | |

**建议**：MVP 先用 IndexedDB，V1.1 评估迁移到 SQLite WASM

## 数据模型

### Product（商品）
```typescript
interface Product {
  id: string;           // UUID
  name: string;         // 商品名称
  price: number;        // 售价
  stock: number;        // 当前库存
  barcode?: string;     // 条码（可选） 自動產生 
  createdAt: Date;
  updatedAt: Date;
}
```

### Transaction（交易）
```typescript
interface Transaction {
  id: string;
  items: TransactionItem[];
  total: number;
  createdAt: Date;
}

interface TransactionItem {
  productId: string;
  name: string;         // 快照商品名称
  price: number;        // 快照售价
  quantity: number;
}
```

## 风险评估与缓解

| 风险 | 影响 | 缓解措施 |
|-----|------|---------|
| IndexedDB 容量限制 | 中 | 定期提醒导出备份；MVP阶段数据量小 |
| iOS PWA 限制 | 低 | 引导用户添加到主屏幕；充分测试 |
| 条码扫描兼容性 | 中 | 使用成熟库（zxing-js），准备手动输入 fallback |
| 浏览器存储被清理 | 中 | 定期提醒用户备份数据 |
| 无网络时无法验证付费 | 高 | 本地许可证文件 + 定期联网验证（宽松策略） |

## 关键决策点

### 决策1：IndexedDB vs SQLite WASM
~~- **当前建议**：MVP 用 IndexedDB，快速验证需求~~ 直接使用 SQlite WASM
- **升级时机**：当用户反馈数据量大、查询慢时，迁移到 SQLite WASM

### 决策2：免费 vs 付费
- **MVP 阶段**：全部免费，收集反馈
- **付费时机**：有 100+ 活跃用户后，推出付费版本

### 决策3：PWA vs 原生 App
- **MVP 阶段**：PWA 足够
- **原生 App 时机**：如果需要调用蓝牙打印机、更深度系统整合时考虑
